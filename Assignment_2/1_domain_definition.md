# ⚽ 지식 도메인 정의서 (Domain Definition)

본 문서는 **2번 과제: MCP 서버 기반 Wiki Tool**이 다루는 지식 도메인과 이를 시각화 및 서빙하는 아키텍처에 대해 정의합니다.

---

## 1. 대상 지식 도메인: 현대 축구 전술 및 선수 프로필 데이터베이스
기존의 기본 주제(CSE-3308 강의 내용)를 탈피하고, 개념 간 상호 관계성이 매우 깊고 복잡한 **"유럽 축구 리그, 클럽, 선수 및 전술 시스템"**을 최종 지식 도메인으로 선정했습니다.

### 무엇을 만들 것인가 (What)
*   **선수(Player)**, **클럽(Club)**, **전술(Tactics)**, **리그(League)** 정보를 담은 다차원 관계형 마크다운 위키 저장소를 구축합니다.
*   가장 널리 알려진 **30개의 대표 키워드**(선수 12명, 클럽 10개, 전술 5개, 리그 3개)를 선정하여 유기적으로 교차 연결(Cross-linking)한 30종의 위키 노드를 컴파일하여 구성했습니다.

### 왜 만들 것인가 (Why)
*   **복잡한 다대다(N:M) 관계성**: 선수는 클럽에 속하고, 클럽은 리그에 속하며, 감독과 선수는 전술을 실행합니다. 이러한 상호작용은 파일들이 서로 연결된 그래프(Graph) 구조일 때 가장 빛을 발합니다.
*   **사전 컴파일(Pre-compilation)의 이점**: 런타임에 다량의 문서를 검색해서 파싱하는 무거운 RAG와 달리, 지식을 미리 구조화된 연결 구조로 컴파일해 놓음으로써 LLM Agent와 인간 모두가 초고속으로 일관성 있는 정보를 탐색할 수 있습니다.

### 어떻게 만들 것인가 (How)
1.  **원시 소스 문서 저장소 (`raw/`)**: 지식 컴파일 대상이 되는 미가공 텍스트 정보가 적재되는 폴더입니다.
2.  **지식 저장소 (`wiki/`)**: 마크다운 파일과 YAML 메타데이터를 활용해 지식을 적재합니다.
3.  **스키마 정의 (`schema/`)**: 위키 지식 노드가 준수해야 하는 메타데이터 규격을 `schema.md`로 제공합니다.
4.  **시각화 도구 및 MCP 서버 (`tools/`)**:
    *   `tools/mcp_server.py`: FastMCP 프레임워크를 기반으로 지식의 조회, 검색, 수정, 검증을 처리하는 API 도구를 제공합니다.
    *   `tools/server.py`: 웹 기반의 프리미엄 대시보드 및 AI 에이전트 브릿지 서버입니다.
5.  **에이전트 운영 하네스 (`harness/`)**: 에이전트의 작동 지침 (`RULES.md`)과 함께 무결성 검증을 담당하는 핵심 SKILL 프로그램 (`skills/audit_syntax.py`)을 운용합니다.

## 2. Wiki Pages 시각화 및 서빙 방식 (Visualization & Serving)
*   **Premium Card-Based Dashboard UI**: 
    *   모든 지식은 역할(Player, Club, Tactics, League)에 맞는 컬러풀한 배지와 카드 형태로 격자 레이아웃에 시각화됩니다.
    *   카드를 클릭하면 상세 보기 모달이 팝업되며, 마크다운 본문이 정제된 스타일로 렌더링됩니다.
*   **유기적 네비게이션**: 
    *   모달 하단에 연결된 지식 링크(`links`)들이 시각적 링크 버튼으로 리스트업됩니다. 
    *   특정 버튼(예: `토트넘 홋스퍼`)을 클릭하면 부드러운 트랜지션과 함께 바로 해당 카드의 상세 정보로 즉시 화면 전환이 이루어집니다.
*   **대화형 에이전트 통합**:
    *   대시보드 우측 영역에 상시 대기 중인 챗봇 패널을 배치하여, 에이전트가 배후에서 MCP 도구들을 호출하는 과정(예: `verify_wiki_health`, `search_wiki` 등)을 실시간 로그 형태로 시각화합니다.

## 3. 도구 활용을 위해 필요한 핵심 기능 정의
사용자가 축구 위키 도구를 효과적으로 활용하기 위해 정의한 필수 기능 명세입니다:
1.  **키워드 다차원 검색 (`search_wiki`)**: 축구 클럽이나 특정 키워드를 검색하면 관련 가중치가 높은 문서들을 추출합니다.
2.  **문서 세부 탐색 (`read_wiki_page`)**: 개별 카드 클릭 또는 에이전트 명령 시 문서를 마크다운 규격으로 실시간 조회합니다.
3.  **지식 확장 및 갱신 (`write_wiki_page`)**: 새로운 선수 이적이나 신규 전술 수립 시 일관된 스키마 규칙을 적용해 문서를 자동 생성하고 버전을 갱신합니다.
4.  **자동 참조 무결성 감사 (`verify_wiki_health`)**: 깨진 링크, 고아 문서(Orphan), 잘못된 YAML 형식을 프로그램적으로 전수 조사하여 위키의 정합성을 수치(Health Score)로 산출합니다.
