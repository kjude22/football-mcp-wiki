    *   **시각화 방식**: `gui/` 정적 파일 레이어와 `tools/server.py` 경량 웹 서버를 구축하여 사용자에게 **카드형 대시보드(Card-Based Dashboard)**와 **에이전트 챗봇 창**을 실시간 통합 렌더링함.
    *   **보안 및 검증**: 새로운 페이지를 작성하는 `write_wiki_page` 도구 내부에서 YAML frontmatter 구조 강제화 및 `verify_wiki_health` 도구를 통해 상대 경로 무결성을 자체 검증함.
*   **리뷰어 피드백 (Reviewer)**: **[평가 점수: 9/10 - APPROVED ✅]**
    *   *결정 사항*: 
        - 제안된 4개의 도구(`search_wiki`, `read_wiki_page`, `write_wiki_page`, `verify_wiki_health`)는 요구사항에 완전히 일치함.
        - 경량화된 표준 웹 기술(HTML, CSS, JS)과 파이썬 기본 `http.server`를 사용함으로써, 외부 의존성(FastAPI/Node 등) 설치 실패 리스크를 극소화하고 안정적인 로컬 서빙이 가능하도록 한 점을 높게 평가함.

## 5. API 속도 제한(429 Rate Limit) 예외 처리 고도화 (Round 4)
*   **날짜**: 2026-06-03 15:05
*   **이슈**: 구글 제미나이 무료 API 플랜은 분당 요청 횟수(RPM) 제한이 15회로 엄격하여, 사용자가 연속으로 챗봇에 질의할 때 429(Quota Exceeded) 오류를 빈번하게 마주치는 현상이 발견됨.
*   **합의된 해결책**: 
    - 429 오류가 발생하더라도 사용자 서비스가 차단되거나 깨지는 대신, **시스템이 자동으로 감지하여 임시 로컬 시뮬레이션 모드로 즉각 전환(Auto Fallback)**되도록 서버 코드를 개선함.
    - 사용자에게는 "API 호출 제한에 도달하여 임시 로컬 모드로 전환되었습니다"라는 명확한 알림과 안내를 제공하여, 챗봇을 통한 위키 문서 검색, 읽기, 쓰기 및 건강 검사 MCP 도구 작동이 끊김 없이 정상 수행될 수 있도록 함.
