# 🤖 PRD 및 에이전트 상세 명세서 (Agent Spec & PRD)

본 문서는 **Gemini AI 모델 및 MCP 도구를 적용한 축구 위키 도구**의 제품 요구사항(PRD)과 위키를 관리하는 인공지능 에이전트 군의 역할 및 권한 규격(SPEC)을 상세하게 기술합니다.

---

## 1. 제품 요구사항 정의 (PRD)

### 1.1 프로젝트 목표
*   사용자 및 인공지능 에이전트가 축구 관련 정보(선수, 클럽, 전술, 리그)를 손쉽게 적재하고, 이들 사이의 유기적인 관계를 카드 형태의 웹 UI로 탐색할 수 있는 통합 지식 도구를 구현합니다.

### 1.2 핵심 사용자 여정 (User Flow)
1.  **지식 탐색**: 사용자는 웹 대시보드에서 선수/클럽/전술/리그 카드 30종을 둘러보며, 카드 클릭 시 나타나는 팝업을 통해 상세 정보와 연결 링크들을 연쇄적으로 클릭하여 탐색합니다.
2.  **AI 에이전트 협업 (Gemini API 기반)**: 사용자가 우측 챗봇 창에 자연어로 질문을 던집니다. (예: "손흥민 정보를 가져오고 소속 클럽인 토트넘 페이지와 연동해줘")
3.  **자율 MCP 도구 구동**: Gemini 모델이 본문의 의도를 파악하여 제공된 MCP 도구(`search_wiki`, `read_wiki_page`, `write_wiki_page`, `verify_wiki_health`)를 직접 자율 선택하여 구동합니다.
4.  **UI 동적 갱신**: 도구 실행 결과로 새 위키 문서가 작성되거나 수정되면, 웹 화면의 카드들이 별도의 새로고침 없이 비동기적으로 추가/갱신됩니다.

---

## 2. 에이전트 권한 및 스펙 정의 (Agent SPEC)

위키 시스템을 편집하고 검증하는 자율 에이전트들의 안전한 운용을 위해 권한 및 허용된 기능 범위를 선언적으로 제약합니다.

### 2.1 에이전트의 역할 분담 및 정의
프로젝트 내부의 가상 에이전트 풀은 3개의 전문화된 역할로 분리되어 상호작용합니다.

| 에이전트 ID | 역할 (Role) | 주요 임무 (Responsibilities) |
| :--- | :--- | :--- |
| **FootyAgent (Coder)** | 지식 작가 및 편집자 | 새로운 지식 요청을 파싱하여 축구 위키 문서 생성 및 기존 문서의 관계성 링크 업데이트 수행. |
| **TacticPlanner (Planner)**| 전술 및 관계 기획자 | 입력된 가공되지 않은 텍스트 데이터를 분석하여 최적의 파일명, 매칭할 태그 리스트, 연결 문서 목록(`links`)을 추출 및 기획. |
| **FootyAuditor (Tester)**  | 무결성 검증 감사관 | 작성된 문서의 YAML 문법, 루트 상대 경로 규칙, 깨진 참조 링크 여부를 `harness/skills/audit_syntax.py` 스크립트를 실행해 검증하고 최종 건강도 점수를 산출. |

### 2.2 에이전트 권한 및 제약 조건 (Harness Policy)
보안과 시스템의 안전을 보장하기 위해 `harness/RULES.md`에 기초한 엄격한 권한 필터를 적용합니다.

*   **1) 파일 시스템 접근 권한 제한 (Sandbox)**:
    *   에이전트에게 허용되는 읽기 및 쓰기 영역은 오직 `raw/`, `wiki/` 디렉토리 하위의 텍스트 및 마크다운 파일로 제한됩니다.
    *   시스템 설정 파일이나 소스 코드 파일(`tools/server.py`, `tools/mcp_server.py`, `gui/*`) 영역에 대한 임의적인 쓰기 및 덮어쓰기 권한은 완전히 **배제**됩니다.
*   **2) 허용된 도구 인터페이스 제한**:
    *   에이전트는 제공되는 4가지 MCP 표준 도구(`search_wiki`, `read_wiki_page`, `write_wiki_page`, `verify_wiki_health`) 인터페이스를 통해서만 지식 데이터베이스에 개입할 수 있습니다.
    *   OS 쉘 명령어 실행이나 외부 네트워크 API 호출 도구 권한은 안전을 위해 완전히 차단됩니다.
*   **3) 메타데이터 규칙 준수 의무**:
    *   모든 쓰기 동작 시, YAML 메타데이터 헤더를 반드시 동반해야 하며 제목(`title`) 필드는 반드시 큰따옴표로 감싸진 문자열이어야 합니다. 규칙 위반 시 Tester 단계에서 감사관(Auditor)에 의해 거부됩니다.
